  <files>.devcontainer/devcontainer.json</files>
  <action>
Create a devcontainer.json that:
1. References the Dockerfile from Task 1: `"image": "..."` or `"build": {"dockerfile": "Dockerfile"}`
2. Sets `"workspaceFolder": "/workspace"`
3. Mounts the repository root at `/workspace` (VS Code/Cursor handles this automatically; devcontainer spec does not require explicit mounts in JSON)
4. Declares VS Code extensions to install in the container:
   - `James-Yu.latex-workshop` (primary LaTeX editor tool)
   - `DavidAnson.vscode-markdownlint` (optional, for README/docs)
5. Declares ports if needed (none for LaTeX, but useful for future PDF server if added): omit for now
6. Sets container user to `vscode` (non-root)
7. Optionally forwards git credentials from host to container: add `"remoteUser": "vscode"` and ensure SSH forwarding via Docker daemon
8. Sets reasonable resource limits in comments (e.g., 4GB RAM, 2 CPUs) for developers to adjust

Structure:
```json
{
  "name": "LaTeX Workspace",
  "build": {
    "dockerfile": "Dockerfile"
  },
  "workspaceFolder": "/workspace",
  "remoteUser": "vscode",
  "customizations": {
    "vscode": {
      "extensions": [
        "James-Yu.latex-workshop"
      ]
    }
  },
  "postCreateCommand": "latexmk --version && chktex --version && latexindent --version",
  "forwardPorts": []
}
```

Also add a comment explaining git credential forwarding for submodules (e.g., private projects).
